## 系统权限控制插件 vivien-permission

使用文档：https://github.com/yoguoer/vivien-permission

​	系统权限控制模块是一个关键组件，它确保系统中的每个用户只能访问其被授权的资源。vivien-permission 插件是一个基于后台管理系统中的路由菜单权限控制系统，通过 vue-router 全局控制后台管理系统的菜单权限。

### 功能

| 功能             | 介绍                                                     |
| ---------------- | -------------------------------------------------------- |
| 菜单路由权限控制 | 通过接口返回权限路由名称，控制当前登录用户的路由权限     |
| 按钮级别权限控制 | 通过接口返回按钮权限列表名称，控制当前登录用户的按钮权限 |
| 单点登录         | 使用当前插件的系统和其他系统相互登录                     |

**单点登录（SSO）：**

- 概念：单点登录是一种身份验证机制，它允许用户在多个关联系统中使用同一套登录凭据（如用户名和密码）进行身份验证。一旦用户在一个系统中成功登录，他/她就可以无缝地访问其他所有关联系统，无需再次输入登录信息。
- 优点：单点登录极大地提高了用户体验，降低了用户的记忆成本和管理员的管理负担。它减少了用户需要记住的多个用户名和密码的数量，同时也简化了管理员对用户账户的管理和维护工作。
- 实现方式：为了实现单点登录，通常需要引入统一身份认证系统。这个系统包括用户目录（存储用户身份信息）、认证服务器（处理登录请求并验证身份）和授权服务器（根据用户身份和权限控制访问）。通过将这些组件与不同的应用集成，可以实现单点登录和统一身份认证功能。

**路由权限控制：**

- 概念：路由权限控制是对系统中不同页面的访问权限进行管理。它基于用户的角色或身份，决定用户可以访问哪些页面或执行哪些操作。
- 功能：路由权限控制通常与角色管理、权限管理等功能相结合。管理员可以为不同的角色设置不同的路由访问权限，从而确保只有具有相应权限的用户才能访问特定的页面或功能。
- 实现方式：在实现路由权限控制时，通常会采用前端路由和后端路由相结合的方式。前端路由负责在用户界面层面控制页面的跳转和显示，而后端路由则负责在服务器端验证用户的访问权限。当用户尝试访问某个页面时，前端路由会检查用户的权限，如果权限不足，则阻止页面跳转或显示相应的提示信息。同时，后端路由也会在服务器端验证用户的访问请求，确保只有具有相应权限的用户才能获取到相应的数据或执行相应的操作。

综上所述，系统权限控制模块通过单点登录和路由权限控制等功能，实现了对用户访问权限的精细化管理。这不仅提高了系统的安全性，也提升了用户体验和管理效率。

## 自定义指令插件 v-permission

​	后台管理系统中，很多操作、视图的显隐都是与职责和角色挂钩的，同样一个列表，对某些用户来说是只读浏览数据，对某些用户来说是可编辑的数据，有些组件对某些用户也是不可见的。v-permission 插件是一个基于Vue3进行封装的自定义指令，在这个插件中，你可以检查传入的系统权限列表和用户拥有的权限列表，来确定用户是否具有某个组件/按钮级别的权限，实现更细粒度的权限控制。

### *功能*

① 指令：有权限时显示 UI，无权限时不渲染。避免出现用户看得见按钮，点击却无反应或出现无权限提示，这种不友好的使用体验。

②方法：可用于一些前置处理，比如进入页面时，根据是否有权限来判断默认渲染哪个组件作为该用户能看到的首页。

使用文档：https://github.com/yoguoer/v-permission.git
